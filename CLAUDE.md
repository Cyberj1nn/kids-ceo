## О проекте

Kids CEO (Malafeeva School) — веб-приложение онлайн-школы, аналог GetCourse. PWA с адаптивной вёрсткой. Платформа для обучения: лекции, инструкции, программы, марафоны, чаты, таблицы ДТП, трекер созвонов, админка.

> Тестирование и деплой в план разработки не включены.

---

## Структура монорепо

```
client/    — React + Vite (TypeScript)
server/    — Node.js + Express (TypeScript)
```

**Инфраструктура:** Ubuntu-сервер на Beget, домен kids-ceo.ru, GitHub + автодеплой

## Архитектурные соглашения

### Принцип разработки

- **Вертикальные слайсы**: каждый блок = миграция БД + API-роуты + UI за один раз
- **Связи между блоками** — создаются при разработке блока, не отдельной фазой

### Backend: структура

- **Слои строго:** `Route → Middleware → Handler → DB-запрос` (Handler не обращается к БД через ORM напрямую, использует функции из `db/queries/`)
- **Валидация** входных данных на уровне роутов
- **API:** REST, JSON
- **Ошибки:** единый формат `{ error: string, details?: any }`
- **Миграции:** SQL-файлы в `server/src/db/migrations/`, выполняются последовательно

### Система ролей (RBAC)

| Роль       | Доступ                                                                 |
| ---------- | ---------------------------------------------------------------------- |
| superadmin | Полный доступ, назначение ролей admin/mentor, управление всей платформой |
| admin      | Контент, пользователи (только роль user), чаты, ДТП, трекер            |
| mentor     | Идентичен admin (разделение семантическое, не по правам)                |
| user       | Просмотр открытых вкладок, заполнение ДТП, участие в чатах             |

### Доступ к вкладкам

- Таблица `user_tab_access` контролирует, какие вкладки видит пользователь
- Middleware `tabAccessCheck` проверяет доступ на каждом запросе к контенту вкладки
- admin/mentor/superadmin имеют доступ ко всем вкладкам без записей в `user_tab_access`

### 11 вкладок платформы

1. Беседа (общий чат)
2. Лекции (9 подкатегорий: бухгалтерия, бизнес-процессы, маркетинг, найм, руководитель, учет, продажи, команда, мышление)
3. Инструкции (8 подкатегорий: команда, маркетинг, продажи, учёт, руководитель, открытие, работа центра, технические вопросы)
4. Подкасты
5. Книги
6. Фильмы
7. Программа Руководитель 360 (6 блоков: Старт сезона, Продажи 1, Клиенты, Команда, Структура, Продажи 2)
8. Марафоны (6 подкатегорий: бизнес-процессы, администратор, команда, найм, супер-ассистент, привлечение клиентов)
9. Личная беседа (приватный чат: пользователь ↔ админы/наставники)
10. ДТП (таблица Достижения/Трудности/Предложения)
11. Трекер созвонов

### Контент

- Вкладки 2–8: rich-text (TipTap) с вложениями — текст, фото, PDF, ссылки, видео (Rutube embed + фоллбэк)
- Вкладки 4–6 (Подкасты, Книги, Фильмы): формат как у лекций, без подкатегорий
- Загрузка файлов: изображения до 10 МБ, PDF до 50 МБ, валидация MIME-типов

### Чаты (Socket.io)

- Общий чат (type=general) — одна комната для всех пользователей платформы
- Личные беседы (type=personal) — по одной комнате на каждого пользователя + все админы/наставники
- Текст, ссылки, базовые эмодзи
- Уведомления (badge) — только для личных бесед

### ДТП (Достижения, Трудности, Предложения)

- Структура: месяц → неделя (1–5) → таблица 4 строки × 7 столбцов-дней
- Строки: Дата | Достижения | Трудности | Предложения
- Автосохранение по onBlur/debounce
- Audit log: каждое изменение записывается (кто, когда, что изменил, старое/новое значение)
- Цветовое кодирование строк (Д — оливковый, Т — красноватый, П — серо-синий)

### Трекер созвонов

- Нумерованные строки (01, 02, 03...)
- Маленькая колонка "Дата" + большая колонка "Краткая информация"
- Заполняется только админами/наставниками, пользователь — только просмотр

### Soft delete

Мягкое удаление через `deleted_at` — физически не удалять записи пользователей и контента.

### Frontend

- **Mobile-first**, адаптив через CSS
- Sidebar (боковая панель) для навигации, на мобильных — гамбургер-меню
- Таблицы ДТП/Трекер на мобильных → карточный вид
- Состояния загрузки: скелетоны, не спиннеры
- Пустые состояния: информативные заглушки

### UI-референс

Визуальный ориентир — сайт skillspace.ru (цветовая гамма, шрифты).
